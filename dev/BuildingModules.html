Building Modules

Overview

You can produce a module in different ways:

  • Build an entire module from scratch.
  • Customize a set of module development templates.
  • Use a pre-existing module. Hundreds of third-party modules are available from these sources:
    • For sale: DNN Store - Some developers provide versions that include the source code.
    • Open source and free:
      • DNN Forge
      • DNN at Codeplex
      • DNN at GitHub

Prerequisites

  • A local DNN development environment

Steps

  1. Decide what type of module you need:
    • A Web Forms module ....
    • A MVC module ....
    • A SPA module ....
  2. Modify the module to implement the functionality you need:
    • Web Forms module
    • MVC
    • SPA
  3. Test your module.

    Tip: Before you start your own tests, try the DNN Extension Verification Service (EVS) to perform the initial compatibility tests.

    • In the module packaging, verifies that
      • That a valid .dnn manifest file exists. Errors are issued, if required sections are missing; warnings, if optional sections are missing.
      • That all files listed in the manifest exist in the package.
      • That all files included in the package are listed in the manifest.
    • In the data layer, verifies:
      • That any SQL scripts in the module are compatible with SQL Azure. If Azure-incompatible SQL scripts are found, EVS generates Azure-compatible versions of those scripts and makes them available in a zip file.
      • That the uninstall script completely removes all objects added by the installation script.
    • In the assemblies, verifies:
      • That no assembly errors exist.

      By examining the assembly references, the EVS also determines the minimum .NET version and minimum DNN version required for the module to work correctly.

  4. Package your module.
