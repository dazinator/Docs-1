Create a Container

Overview

Containers allow designers to seamlessly integrate modules from many different developers into a cohesive site design. Containers can also include visual elements that allow site users to interact with the module.

  • You can have a default container, which would be overridden by individual pages or modules if they provide their own style definitions.
  • You can mix and match any container with any skin or use different containers in a single page.

Every container includes the Module Action Menu which administrators and editors use for configuring the module.

Steps

  1. Create a new HTML file for the container.

    Note:

    • The container must contain at least one pane.
    • Exactly one of the panes must be called ContentPane.
    • All panes must be defined as a server control by adding runat="server" to the element.
    • DNN recognizes only the folowing HTML elements as valid panes: <td> (table cells), <div>, <p>, and <span>.

    Example: A very basic container.

    <div id="wrapper"> <div id="ContentPane" runat="server"></div> </div>

  2. (Optional) Add theme objects to your container for a more dynamic page.

    The following theme objects are relevant to containers:

    ACTIONS
