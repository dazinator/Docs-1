Build an HTML Skin

Overview

The most basic DNN skin configuration is comprised of one layout template. However, you can also create a master CSS file for all layout templates or one CSS file for each layout template.

Prerequisites

  • A super user (host) account. Super user privileges are required to upload skins, because skins can include executables. The super user also converts the skin from HTML to ASCX. (An administrator account is sufficient to apply skins and containers to a site.)

Steps

  1. Create a layout template for the skin as an HTML file.
    1. In your DNN installation, create a new folder for your skin under Portals\_default\Skins. The new folder name becomes the name of the DNN skin.
    2. In the new folder, create a new layout template file called index.html.

      Rules:

      • The layout template must contain at least one pane.
      • Exactly one of the panes must be called ContentPane.
      • All panes must be defined as a server control by adding runat=”server” to the element.
      • DNN recognizes only the folowing HTML elements as valid panes: <td> (table cells), <div>, <span>, and <p>.

      Example: A very basic layout template.

      <div id="wrapper"> <div id="ContentPane" runat="server" ></div> </div>

      Example: A layout template with several panes, a header, and a footer.

      <div id="wrapper"> <div id="header"></div> <div id="sidebar"> <div id="feed"> <a class="feed-button" href="#">&nbsp;</a> </div> <div id="SidebarPane" class="SidebarPane" runat="server"></div> <div id="sidebar-bottom">&nbsp;</div> </div> <div id="content"> <div id="BannerPane" class="BannerPane" runat="server"></div> <div id="ContentPane" runat="server" ></div> </div> <div id="footer"></div> </div>

      Tip: Add a class attribute to each of the pane controls, because ASP.NET will modify the control names when the page is rendered.

      You can add these skin objects using the HTML object and param tags. In the object tag:

      • The codebase attribute specifies the skin object to be rendered.
      • The codetype attribute must be dotnetnuke/server for all skin objects.

      Example: Adding the LOGIN skin object.

      <div id="login_style" class="user"> <object id="dnnLOGIN" codebase="LOGIN" codetype="dotnetnuke/server"> <param name="CssClass" value="user" /> </object> </div>

  2. Convert the layout template from HTML to ASCX.
    1. In the DNN administration page, go to Admin > Skins.
    2. Choose your new skin in the Skins dropdown, then click Parse Skin Package.

    RESULT: DNN converts the very basic layout template (index.html) to a new file (index.ascx) similar to this:

    <%@ Control language="vb" AutoEventWireup="false" Explicit="True" Inherits="DotNetNuke.UI.Skins.Skin" > <div id="ContentPane" runat="server" ></div>

    NOTE: The conversion only goes in one direction (from HTML to ASCX).

  3. Create a CSS file for each layout template, or create a single master CSS file to be used for all layout templates.
    1. Create a CSS file in the same folder as the layout template.
    2. For every section (
      ) that contains a skin object in the layout template, add a style definition in the CSS file. The style definition must start with # followed by the div id. Example:
    3. Test the skin on a webpage.
      1. Create a new web page.
      2. Edit the page settings and go to Advanced Settings.
      3. In the Page Skin section, select Host and choose one of the layout templates from your new skin.
        Note: When a layout template is added to the page, the associated CSS file is automatically added.
      4. Click Update.
      5. View your page in different browsers to verify that the new skin was applied correctly.
    4. If the page is not rendering properly in some browsers, you can force the skin to be rendered in XHTML 1.0 Transitional.
      1. Create a doctype XML file with the same name as your layout template file.
        Example: If your layout template file is called MyLayout.html, give your doctype file the name MyLayout.doctype.xml.
      2. Enter the following in your doctype file: <SkinDocType> <![CDATA[<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">]]> </SkinDocType>
    5. Package your DNN skin.
