Create a Layout Template in HTML Format

Prerequisites

  • A super user (host) account. Super user privileges are required to upload skins, because skins can include executables. The super user also converts the skin from HTML to ASCX. (An administrator account is sufficient to apply skins and containers to a site.)

Steps

  1. In your DNN installation, go to Host > File Management.
  2. In your DNN installation, create a new folder for your skin under Skins\_default. The new folder name becomes the name of the DNN skin.
  3. In the new folder, create a new layout template file called index.html.

    Note:

    • The layout template must contain at least one pane.
    • Exactly one of the panes must be called ContentPane.
    • All panes must be defined as a server control by adding runat="server" to the element.
    • DNN recognizes only the folowing HTML elements as valid panes: <td> (table cells), <div>, <span>, and <p>.

    Example: A very basic layout template.

    <div id="wrapper"> <div id="ContentPane" runat="server"></div> </div>

    Example: A layout template with several panes, a header, and a footer.

    <div id="wrapper"> <div id="header"></div> <div id="sidebar"> <div id="feed"> <a class="feed-button" href="#">&nbsp;</a> </div> <div id="SidebarPane" class="SidebarPane" runat="server"></div> <div id="sidebar-bottom">&nbsp;</div> </div> <div id="content"> <div id="BannerPane" class="BannerPane" runat="server"></div> <div id="ContentPane" runat="server"></div> </div> <div id="footer"></div> </div>

    Tip: Add a class attribute to each of the pane controls, because ASP.NET will modify the control names when the page is rendered.

  4. (Optional) Add skin objects to your layout template for a more dynamic page. A default DotNetNuke installation includes the following skin objects:
    dnnBANNER
    dnnBREADCRUMB
    dnnCOPYRIGHT
    dnnCURRENTDATE
    dnnDOTNETNUKE
    dnnHELP
    dnnHOSTNAME
    dnnLANGUAGE
    		 dnnLEFTMENU
    dnnLINKS
    dnnLOGIN
    dnnLOGO
    dnnMENU
    dnnNAV
    dnnPRIVACY
    dnnSEARCH
    		 dnnSOLPARTMENU
    dnnSTYLES
    dnnTAGS
    dnnTERMS
    dnnTEXT
    dnnTREEVIEW
    dnnUSER

    You can add these skin objects using the HTML object and param tags. In the object tag,

    • The codebase attribute specifies the skin object to be rendered.
    • The codetype attribute must be dotnetnuke/server for all skin objects.

    Example: To add the dnnLOGIN skin object,

    <div id="login_style" class="user"> <object id="dnnLOGIN" codebase="LOGIN" codetype="dotnetnuke/server"> <param name="CssClass" value="user"/> </object> </div>
